2026-04-18, WebSocket resiliency: fixed three bugs in ChatInterface.jsx that the earlier "connection lost" patch left behind. (1) all WS event handlers are nulled on the old socket before `close()`, preventing a late-firing `onopen` from clobbering the new socket's heartbeatRef. (2) auto-retry of pendingMessageRef removed: the server may have already processed the message, resending would double-bill. The 30s loadingTimeout surfaces a user-facing "timed out" prompt instead. (3) "Connection lost" banner is now debounced by 1.5s to stop flicker on brief reconnects (token rehydrate, normal hiccup). Added docs/crash_test_results.md scaffold with three new resilience tests (network interruption mid-stream, browser matrix, auth churn) that extend the three in DS 440/resilience_testing_report.docx without burning API quota.
